# 第7章：集合（Collections）

集合是表示一组对象的对象。例如，栈（stack）是一种集合，集合（set）和向量（vector）也是。Java 定义了一个 `Collection` 接口，规定了任何集合应该提供的操作，包括添加和删除元素的操作，以及获取集合中元素数量的操作。

这个接口非常通用，例如它不规定元素的删除顺序，也不规定是否允许重复。Java 定义了更具体的接口来更完整地规定这些事项并添加更多方法，包括 `Queue`、`List` 和 `Set` 接口。

## 7.1. 实现类

目前为止我们只讨论了接口。每个接口有若干个具有不同特性的实现。例如，`HashSet` 用哈希表实现了 `Set` 接口，使其核心方法 `add`、`remove`、`size` 和 `contains` 都能达到常数时间性能，但无法保证迭代时元素出现的顺序。而 `TreeSet` 则用平衡树实现了该接口，因此可以保证顺序，但只能保证核心方法以对数时间运行。

选择某种集合的实现时，可以通过阅读文档来了解各种选项的特性。

## 7.2. Java 集合框架

我们将各种接口、其实现、在集合上操作的静态方法以及一些附加基础设施统称为"Java 集合框架（Java Collections Framework，JCF）"。

下面我们来了解几个常用的实现，感受一下它们的用法。

### 7.2.1. List（列表）

Java 的 `List` 与 Python 的 `list` 类型类似，会根据需要自动扩容和缩容。`ArrayList` 是其中一种实现，提供通过索引快速访问元素的能力。

```java
public static void main(String[] args) {
    // 声明并初始化一个 String 类型的 ArrayList：
    ArrayList<String> csc207team = new ArrayList<>();

    csc207team.add("Juanita");
    csc207team.add("Amelie");
    csc207team.add("Abhinav");
    csc207team.add("Menghan");

    // ArrayList 有一个 toString 方法可以漂亮地打印列表。
    System.out.println(csc207team);

    // 这是不允许的，因为我们指定了元素类型为 String：
    // csc207team.add(5);

    // 不需要类型转换，编译器知道 get 会返回一个 String，
    // 因为这是一个 String 类型的 ArrayList。
    csc207team.get(0).charAt(0);

    // 一些其他有用的方法。ArrayList 还有更多——请查阅文档！
    System.out.println(csc207team.size());
    System.out.println("Is alex in csc207team? " +
    csc207team.contains("alex"));

    // 我们可以有任何有效 Java 对象类型的 ArrayList，
    // 即任何内置或用户定义的对象。但不能是基本类型，所以下面不行！
    // ArrayList<int> wontWork = new ArrayList<int>();

    // 但可以通过包装类将基本类型放入 ArrayList。
    List<Integer> intList = new ArrayList<>();

    // 我们可以利用自动装箱来避免手动构造包装类实例。
    // 因此不必这样写：
    // intList.add(new Integer(42));
    // 直接这样就可以：
    intList.add(42);
    System.out.println(intList);

    // 也可以自动拆箱。从 ArrayList 取出 Integer 对象并得到 int，只需：
    int x = intList.get(0);
    System.out.println(x);
}
```

`List` 的所有元素必须是对象而非基本类型，且必须是相同类型。不过，我们可以利用继承来允许混合类型。例如，如果我们指定所有元素都是 `Object` 类型，则没有任何限制。

### 7.2.2. Set（集合）

如你所料，`Set` 不允许重复元素，也没有元素位于某个特定位置的概念。`TreeSet` 是其中一种实现。

```java
public static void main(String[] args) {

    // 声明一个 String 类型的 TreeSet，并尝试添加一些元素。
    TreeSet<String> s = new TreeSet<>();
    s.add("hello");
    s.add("silly");
    // 可以查看 add 的返回值来判断操作是否成功。
    System.out.println(s.add("goodbye!"));
    // 不能第二次或第三次添加同一个 String。
    System.out.println(s.add("silly"));
    System.out.println(s.add("silly"));

    // TreeSet 有一个 toString 方法可以漂亮地打印集合。
    // 元素出现的顺序可能是任意的。
    System.out.println(s);

    // TreeSet 实现了 Iterable 接口，保证提供 hasNext 和 next 方法。
    // 这里我们用它来迭代集合，将所有单词拼接成一个 String。
    String allWords = "";
    Iterator<String> it = s.iterator();
    while (it.hasNext()) {
        allWords += it.next();
    }
    System.out.println(allWords);

    // 因为 TreeSet 实现了 Iterable，我们可以改用增强型 for 循环，更简洁！
    allWords = "";
    for(String word: s) {
        allWords += word;
    }
    System.out.println(allWords);
}
```

### 7.2.3. Map（映射）

`Map` 接口类似于 Python 的 `dict` 类型，将键映射到值，每个键只能关联一个值。与 Python 不同的是，键不必是不可变的。但如果你在将一个对象添加为键之后又对其进行了修改，`Map` 的行为将是未定义的，因此必须非常清楚自己在做什么。详细信息请查阅文档。

`HashMap` 是 `Map` 的一种实现，基于——你猜对了——哈希表！

```java
public static void main(String[] args) {
    // 声明并初始化一个从 String 到 Integer 的 Map。
    // 注意，HashMap 的泛型类型需要两个参数：
    // 一个是键的类型，一个是值的类型。
    Map<String, Integer> myMap = new HashMap<>();

    // 我们可以使用自动装箱将 Integer 值放入 HashMap。
    myMap.put("csc", 207);
    myMap.put("bio", 120);
    myMap.put("his", 150);
    myMap.put("ant", 100);
    System.out.println(myMap.get("csc"));

    // "mat" 不是键，所以返回 null。
    System.out.println(myMap.get("mat"));

    // HashMap 还有很多其他方法——详细信息请查阅文档。
}
```

`HashMap` 的基本操作（`get` 和 `put`）可以达到常数时间性能，非常出色。不过，这依赖于哈希表在键值对的添加和删除过程中维持某些特性。构造 `HashMap` 时，你可以控制数据结构的参数，有助于确保这些特性得以维持。在 CSC263 中你将深入学习哈希表，届时就能做出更好的选择——值得期待！
