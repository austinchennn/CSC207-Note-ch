# 第6章：泛型（Generics）

泛型是程序员将类或方法所处理的类型泛化的一种方式。它让我们可以将同一份代码复用于各种输入类型，而无需频繁进行类型转换。

例如，考虑 Java 内置的 `List<T>` 接口。该接口规定了一个类存储泛型类型 `T` 对象列表的含义。当我们声明一个引用整数列表的变量时，将类型指定为 `List<Integer>`。

下面是一个使用泛型的简单示例：

```java
// 右侧不需要 <Integer>，因为它是隐式的
ArrayList<Integer> ma = new ArrayList<>();
ma.add(1);
Integer my_item = ma.get(0) + 5;
```

`<Integer>` 指定我们使用的是只包含 `Integer` 的 `ArrayList`，无需进行任何类型转换。

在本课程中，我们主要会使用已经用泛型编写好的代码，比如上面的 `ArrayList<T>`。当然，也可以在自己定制的类中使用泛型，如下所述，但在本课程中通常不需要这样做。

## 6.1. 自定义泛型类

简而言之，定义泛型类的语法如下：

```java
public class Box<T> {
    private T item;

    public void set(T item) {
        this.item = item;
    }

    public T get() {
        return item;
    }
}
```

在这个例子中：

- `Box<T>` 是一个泛型类，`T` 是类型参数。
- 可以为不同类型创建 `Box` 的实例，例如 `Box<Integer>` 或 `Box<String>`。
- 类型 `T` 在类被实例化时会被实际类型替换。

使用该泛型类的示例可能如下所示：

```java
Box<Integer> intBox = new Box<>();
intBox.set(123);
Integer value = intBox.get();

Box<String> strBox = new Box<>();
strBox.set("Hello");
String text = strBox.get();
```

## 6.2. 有界类型参数（Bounded Type Parameters）

你也可以使用**有界类型参数**来**限制**泛型类或方法可以使用的类型。当你的代码依赖于某些特定行为或接口时，这非常有用。

例如，Java 的 `Collections.sort()` 方法要求列表中的元素实现 `Comparable` 接口，以便能够排序。

下面是如何定义一个泛型方法，对保证可比较的元素列表进行排序：

```java
public static <T extends Comparable<T>> void sortList(List<T> list) {
    Collections.sort(list);
}
```

在这个例子中：
- `<T extends Comparable<T>>` 表示 `T` 必须是实现了 `Comparable<T>` 的类型。
- 这确保了 `sortList` 方法可以安全地对列表中的元素调用 `compareTo()`。

如果你尝试用一个未实现 `Comparable` 的类型调用 `sortList()`，会得到一个编译时错误。这正是泛型设计的目的之一。

事实上，我们的 `sortList()` 比实际需要的限制略严一些。如果查看 `Collections.sort` 的签名，实际上是：

```java
public static <T extends Comparable<? super T>> void sort(List<T> list)
```

`<T extends Comparable<? super T>>` 表示 `T` 必须是可以与**其自身或其任何超类型**进行比较的类型。

`?` 是一个通配符，代表未知类型。在 `Comparable<? super T>` 中，它表示我们不关心确切的类型是什么，只要它是 T 的超类型即可。这让该方法可以适用于更广泛的类型，尤其是在涉及继承时。

如果签名只是 `<T extends Comparable<T>>`，在如下情况下就会失败：

```java
class Animal implements Comparable<Animal> { ... }
class Dog extends Animal { ... }

List<Dog> dogs = new ArrayList<>();
Collections.sort(dogs); // 使用 Comparable<? super T> 时可以正常工作
```

这之所以有效，是因为 `Dog` 是 `Animal` 的子类型，而 `Animal` 实现了 `Comparable<Animal>`。使用 `? super T` 确保了这种继承关系被允许。

这种设计使 `sort()` 方法在实际代码中能与更广泛的类型兼容。

如果感兴趣，可以在 Java 文档中阅读关于泛型的更深入解释，也可以查看[这里](https://docs.oracle.com/javase/tutorial/java/generics/index.html)的教程。

> 在底层，Java 泛型使用了一种叫做**类型擦除（type erasure）**的技术。这意味着泛型类型在编译时会被替换为 `Object`，编译器会在必要的地方插入类型转换。因此，尽管泛型在编译时为我们提供了类型安全，JVM 在运行时并不保留泛型类型信息——它只看到普通的类和 `Object` 引用。这就是为什么不能做 `new T()` 或检查 `instanceof T` 的原因——运行时没有类型信息。
