# 第3章：类之间的关系

## 3.1. 继承

你可能还记得，在 Python 中，我们可以通过以下方式从另一个类继承方法：

```python
class Child(Parent):
    ...
```

上面的语法将 `Child` 类定义为 `Parent` 类的子类。

Java 中的继承方式类似，但不使用括号，而是使用 `extends` 关键字：

```java
class Child extends Parent {
    ...
}
```

这意味着 `Child` 类继承了 `Parent` 中定义的所有方法和变量，并且 `Child` 是 `Parent` 的一个实例。

在本章中，父类/子类（parent class/child class）和超类/子类（superclass/subclass）这两对术语可以互换使用。

### 3.1.1. 抽象类

抽象类是不应该被直接实例化的类。在 Python 中，我们通过让方法抛出 `NotImplementedError` 来表示它是抽象方法，任何非抽象的子类都必须实现该方法。

在 Java 中，我们使用 `abstract` 关键字来表示一个类是抽象的：这强制规定不应该创建该类的实例，即使类中没有抽象方法！我们也用 `abstract` 关键字来标记抽象方法。

```java
abstract class AbstractClass{
    abstract void something();    // 抽象方法没有方法体！
}
```

任何继承了抽象类的非抽象类，都必须实现其中_所有_抽象方法的方法体：

```java
class NonAbstract extends AbstractClass{
    void something(){
        ...   // 方法体在这里！
    }
}
```

### 3.1.2. 重写方法

在 Python 中，我们可以通过在子类中重新定义父类的方法来覆盖它。Java 中做法相同，但还需要加上 `@Override` 注解，用来告知编译器该方法打算覆盖超类中的某个方法。虽然注解不是强制要求，但加上它有助于防止错误（例如方法名拼写错误、遗漏参数等）。

例如，如果有以下父类：

```java
class Parent {
    void something(){
        ...    // 一些方法体！
    }
}
```

我们可以这样重写 `something` 方法：

```java
class Child extends Parent {
    @Override
    void something(){
        ...    // 我们新的方法体！
    }
}
```

## 3.2. 接口（Interface）

在 Java 中，一个类只能继承（extends）一个类：只有一个父类。但有时候，我们希望为类描述更多行为，而仅靠一个父类是不够的。

例如，假设我们正在编写一个模拟植物的程序。我们有一个 `Plant` 类，所有 `Plant` 对象都能 `breathe`（呼吸）和 `grow`（生长）。还可以有 `Wheat`（小麦）和 `Flower`（花）等子类，以及它们各自的子类。但假设我们想表示某些植物对人类来说是可食用的：例如 `Corn`（玉米）可以有一个 `eat` 方法，`Basil`（罗勒）也可以。但不是所有植物都可食用，所以不能将这个方法添加到 `Plant` 中。我们*可以*定义一个 `EdiblePlant` 类，但那样还需要 `EdibleWheat`、`EdibleFlower` 等等——这不是一个很清晰的解决方案。接口提供了另一种设计思路。

Java 中的**接口（interface）**定义了类能做什么的契约，而不指定它是如何做的。它用于定义跨越潜在无关类的共同行为。这类似于抽象类，但根本区别在于：一个类可以实现任意数量的接口。

接口中：

- 所有方法默认是 `public` 且**抽象（abstract）**的，除非另有标记。你可以在自己的接口中定义方法，并观察 IntelliJ 在添加各种关键字时给出什么警告。
- 接口中的变量默认是 `public`、`static` 和 `final` 的。接口不能有实例变量——可以在 IntelliJ 中试一试看会发生什么。
- 实现接口的类必须为其抽象方法提供实现。

最初，Java 的接口只能包含抽象方法。但自 **Java 8** 起，接口还可以包含：

- `default` 方法——带有方法体的方法，允许在不修改已有实现类的前提下向接口添加新功能，实现向后兼容。
- `static` 方法——适合放置与接口相关的工具性行为。

在我们的例子中，可以定义一个 `Edible` 接口：

```java
interface Edible {
    // 任何实现 Edible 的类都必须实现这个方法
    void eat(); // 注意，不需要写 abstract 或 public 关键字！
}
```

要定义一个带有 `default` 方法 `wash` 的 `Washable` 接口，必须在定义时使用 `default` 关键字：

```java
interface Washable {
    // 提供基本实现的默认方法
    default void wash() {
        System.out.println("正在清洗可食用物品...");
    }
}
```

要同时使用这两个接口，需要使用 `implements` 关键字：

```java
class Corn extends Plant implements Edible, Washable {
    void eat() {
        ...    // 我们的实现！
    }
    
    // 重写默认的 wash 方法
    @Override
    public void wash() {
        System.out.println("正在彻底清洗玉米...");
    }
}
```

如果一个类实现了多个接口，它们在 `implements` 关键字后用逗号分隔。

我们可以实现任意多个接口！此外，接口也可以用 `extends` 继承其他接口。

举个例子，某些食物可以蒸，我们可能想要一个 `Steamable` 接口。由于这些食物也是可食用的，可以这样做：

```java
interface Steamable extends Edible {
    void steam();
}
```

任何 `implements Steamable` 的类都必须同时拥有 `steam` 和 `eat` 方法！

## 3.3. super

在 Python 中，我们可以用 `super()` 来引用父类中的方法。例如，使用 `super().__init__()` 调用父类构造函数，或用 `super().method()` 调用父类方法。

Java 中有 `super` 关键字，功能类似。如果想调用父类的构造函数，使用 `super()`，如果需要传参则写 `super(a, b, c)`。如果想调用父类的方法，则使用 `super.method()`！

注意 Python 和 Java 的区别：Python 用 `super()`（带括号），Java 用 `super`（不带括号）！

### 3.3.1. 构造函数中的 super

在继承另一个类时，Java *要求*子类的构造函数中必须调用其超类的构造函数，并且这个调用*必须*是第一件事。如果子类构造函数中没有显式调用构造函数，Java 会*隐式*调用 `super()`。

例如，以下代码：

```java
class Child extends Parent {
    int attribute1;
    int attribute2;

    public Child(int a, int b) {
        // 这里没有 super() 调用，但它是隐式存在的！
        this.attribute1 = a;
        this.attribute2 = b;
    }
}
```

等价于：

```java
class Child extends Parent {
    int attribute1;
    int attribute2;

    public Child(int a, int b) {
        super();  // 如果 Parent 没有无参构造函数，会发生什么？
        this.attribute1 = a;
        this.attribute2 = b;
    }
}
```

如果 `Parent` 没有无参构造函数，编译时就会报错。Java 只会在父类中唯一存在的构造函数是无参构造函数时才会隐式调用它，即在我们的意图没有歧义的情况下。在 IntelliJ 中开发时，为子类编写构造函数时会看到有用的错误提示。

如果父类有一个无参构造函数，并且我们不需要在子类中执行额外的初始化操作，甚至可以省略子类中的构造函数。Java 会在创建子类实例时隐式调用父类构造函数。

## 3.4. 多态（Polymorphism）

**多态**是指一个对象可以以多种形式出现的能力。在 Java 中，这意味着一个对象可以被视为其自身类、任何超类或其实现的任何接口的实例。

例如，如果有以下代码：

```java
class Dog extends Canine implements Domesticatable { ... }
```

一个 `Dog` 对象同时是：
- `Dog`
- `Canine`
- 可能是 `Animal`（如果 `Canine` 继承了 `Animal`）
- `Object`（回想一下，`Object` 在继承层次的顶端）
- `Domesticatable`（因为 `Dog` 实现了这个接口）

这意味着 `Dog` 实例对_所有_这些类型的 `instanceof` 检查都会返回 `true`：

```java
Dog d = new Dog();
System.out.println(d instanceof Dog);              // true
System.out.println(d instanceof Canine);           // true
System.out.println(d instanceof Animal);           // true
System.out.println(d instanceof Object);           // true
System.out.println(d instanceof Domesticatable);   // true
```

> **注意：** Java 中的 `instanceof` 并不严格限于父/子类关系。
> 如果一个对象可以被**安全地转换**为某种类型，则认为它是该类型的实例。
> 这包括接口——实现了某个接口的对象，对该接口的 `instanceof` 检查会返回 `true`。

多态通过允许对象按其能力而非确切类型被对待，实现了灵活且可复用的代码。

以下是一个展示多态的例子：

```java
Animal[] animals = {new Cat(), new Dog(), new Axolotl()};

for (Animal a : animals){
    a.eat();    // 这一行中的 'a' 可以是各种类型！
}
```

即使有多种类型的对象，我们也能以统一的方式处理它们。如果没有多态，就无法写出这样简洁清晰的循环！许多设计工作归根结底在于定义正确的抽象，使代码编写方式更灵活、更具扩展性且易于推理。

通过面向接口或超类编程，而不是面向具体实现，我们的代码能够与具有共同行为的各种对象一起工作。这不仅减少了重复，也更容易引入新类型而无需改变现有逻辑——这是可维护软件设计的关键原则。

## 3.5. 类型转换（Casting）

如我们所学，与 Python 不同，Java 中所有变量都有声明的类型。我们通常会希望在类型之间进行转换：有时更通用的超类类型更适合某种情况，而子类类型在另一种情况下更合适。

**类型转换（Casting）**是指我们显式地将一个引用的类型更改为另一个类型，通常是为了访问更通用类型中不可用的特定功能。例如，如果我们有一个 `Animal` 类型的引用，但知道它实际上引用的是一个 `Dog` 对象，就可以将其转换为 `Dog` 类型，从而调用只存在于 `Dog` 类中的方法。

```java
Animal a = new Dog(); // 狗是动物，所以这个赋值合法
((Dog) a).bark(); // 转换为 Dog 以访问 bark()
```

类型转换只有在实际对象是目标类型的实例时才安全；否则会在运行时抛出 `ClassCastException`。

上面的例子中实际上有两种类型转换。第一行进行的是**向上转型（upcasting）**：将子类对象赋给超类引用。这是**隐式**的，总是安全的，因为每一只 `Dog` 都是 `Animal`。第二行进行的是**向下转型（downcasting）**：从超类类型转换为子类类型。只有当实际对象是子类的实例时才安全，应该用 `instanceof` 检查后再转换，以避免运行时错误。

使用 `instanceof` 在转换前检查的方式如下：

```java
if (a instanceof Dog) {
    ((Dog) a).bark();
}
```

这确保了转换是合法的，避免了运行时错误。

### 3.5.1. 基本类型的转换

我们也可以对某些基本类型进行转换，例如将 `int` 转换为 `double`，反之亦然：

```java
int x = 1;
double y = 1.1;
double double_x = (double) x;
int int_y = (int) y;
```

对**对象**进行类型转换时，本质上是"重新标注"引用的类型——我们告诉编译器将该对象视为不同的类型，而不改变对象本身。实际数据保持不变。

但对**基本类型**进行转换时，我们是在转换值本身，这可能导致**精度损失**或**截断**。例如，将 `double` 转换为 `int` 会丢弃小数部分：

```java
double y = 3.7;
int truncated = (int) y; // 截断为 3
```

这种转换在某种意义上是**不可逆的**——原始精度丢失了，无法从 `int` 中恢复那个 `.7`，除非回头看原始的 `y`。当然，变量 `y` 本身仍然是 3.7。

基本类型转换可以是**隐式的**（如 `int` 到 `double`）或**显式的**（如 `double` 到 `int`），取决于是否存在数据丢失的风险。

## 3.6. Comparable 接口

### 3.6.1. 实现 Comparable 可以排序

在 Python 中，我们有 `list.sort()` 方法和 `sorted()` 函数来对列表排序。类似地，Java 提供了 `sort` 方法，可以对 `int` 数组或任何其他基本类型的数组进行排序。实际上，`sort` 是重载的：有一系列 `sort` 方法，每一个处理一种基本类型。它们都以静态方法的形式定义在 `Arrays` 类中。

下面我们用接受 `int` 数组的 `sort` 方法：

```java
int[] ages = {10, 24, 3, 45, 83, 9};
Arrays.sort(ages);
```

很好！但如果我们想对其他类型的对象进行排序呢？例如，Java 提供了一个 `MonthDay` 类，可以记录日期中的月份和日期部分（它和许多其他时间相关的类都定义在 `java.time` 包中）。如果我们想对 `MonthDay` 对象数组排序呢？还有 `File` 对象数组和 `Double` 对象数组？Java 的设计者可以设计一个接受 `Object` 数组的通用排序方法，如：

```java
public static void sort(Object[] a)
```

但 `sort` 要完成工作，必须能够比较数组中的元素以确定顺序。它不能直接用 `<` 等运算符来比较两个 `MonthDay` 实例，这些运算符不接受 `MonthDay` 实例作为操作数。

因此，`sort` 方法要求数组中的所有元素都必须实现 `Comparable` 接口。该接口要求实现它的类定义以下方法：

```java
int compareTo(T o)
    将此对象与指定对象进行顺序比较。
    当此对象小于、等于或大于指定对象时，
    分别返回负整数、零或正整数。
```

在 Python 中有类似的概念：如果实现了 `__lt__` 方法，就可以比较和排序对象！Java 只是略有不同，返回的是负整数、零或正整数，而不是 `True` 或 `False`。

回到 `MonthDay` 的例子：`MonthDay` 类的作者定义了这个方法，并声明该类**实现**了 `Comparable` 接口。这意味着如果我们将 `MonthDay` 对象数组传给 `sort`，它*保证*能使用 `compareTo` 来为它们排序。以下是利用这一功能的示例代码：

```java
public static void main(String[] args) {
    // 创建一些 MonthDay 对象并放入数组。
    // 这个类没有提供构造函数，而是通过调用静态方法 "of" 来创建实例。
    MonthDay md1 = MonthDay.of(1, 5);
    MonthDay md2 = MonthDay.of(7, 24);
    MonthDay md3 = MonthDay.of(7, 24);
    MonthDay md4 = MonthDay.of(1, 28);
    MonthDay md5 = MonthDay.of(2, 14);
    MonthDay[] dates = {md1, md2, md3, md4, md5};

    // 因为 MonthDay 实现了 Comparable，我们可以调用依赖它的 sort：
    Arrays.sort(dates);
}
```

`sort` 方法还有另一个要求：数组中的所有元素必须是**相互可比较的（mutually comparable）**。这防止了我们尝试对一个混有 `DateTime` 对象和 `File` 对象的数组排序——这些对象在各自类*内部*是可比较的，但在类*之间*则不行（除非它们的类共享一个实现了 `Comparable` 的父类）。

### 3.6.2. 实现 Comparable 可以进行比较

如果我们只是想比较一个 `MonthDay` 与另一个 `MonthDay`，也可以这样做：

```java
System.out.println(md1.compareTo(md2)); // -6
System.out.println(md2.compareTo(md1)); //  6
System.out.println(md2.compareTo(md3)); //  0
```

对任何实现了 `Comparable` 的类都是如此，包括许多内置类，如 `String`、`File`、`Integer` 和 `Double`。

### 3.6.3. 让自己的类实现 Comparable

考虑以下类：

```java
class Review {
    /**
     * 对书籍或电影等内容的评论。
     */

    // === 类变量 ===

    // 该评论所针对的条目名称。
    String item;
    // 该评论的数字评分，0 到 100 之间。
    private int rating;
    // 评论的文字部分。
    private String text;
    // 该评论获得的点赞数。
    private int likes;

    public Review(String item, int rating, String text) {
        this.item = item;
        this.rating = rating;
        this.text = text;
        this.likes = 0;
    }

    public String toString() {
        return this.item + " (" + this.rating + "): " +
            this.text + "; likes = " + this.likes;
    }

    /**
     * 为该评论点赞。
     */
    public void like() {
        this.likes += 1;
    }
}
```

要使这个类的实例可比较，需要做两件事。首先，必须实现 `compareTo` 方法。如何比较两个评论由我们决定。以下是一个基于评分的实现：

```java
    /**
     * 将此对象与指定对象进行顺序比较。
     *
     * 当此对象小于、等于或大于指定对象时，
     * 分别返回负整数、零或正整数。
     *
     * @param other 要比较的对象。
     * @return 当此对象小于、等于或大于指定对象时，
     *         返回负整数、零或正整数。
     */
    @Override
    public int compareTo(Review other) {
        if (this.rating < other.rating) {
            return -1;
        } else if (this.rating > other.rating) {
            return 1;
        } else {
            return 0;
        }
    }
```

其次，必须更改类的声明，以表明我们已满足实现 `Comparable` 接口的要求：

```java
class Review implements Comparable<Review> {
```

注意我们写的是 `Comparable<Review>` 而不仅仅是 `Comparable`。`Comparable` 接口通过泛型来指定，我们稍后会详细讨论。现在只需知道，我们需要指定声明可以与之比较的对象类型。由于我们希望将 `Review` 类的实例与其他 `Review` 对象进行比较，所以使用这种语法。

现在实现了 `Comparable` 之后，我们可以做与 `MonthDay` 类似的事情：

```java
public static void main(String[] args) {
    Review r1 = new Review("Emoji Movie", 10,
        "Cinematic malware");
    Review r2 = new Review("Dunkirk", 95,
        "Gifted ensemble cast and masterful direction");
    Review r3 = new Review("Spider Man: Homecoming", 95,
        "A fun adventure");
    Review r4 = new Review("My Neighbour Totoro", 99,
        "A work of art");
    Review r5 = new Review("Despicable Me 3", 60,
        "Zany but scattershot humour");

    System.out.println(r1.compareTo(r2)); // -1
    System.out.println(r2.compareTo(r1)); // 1
    System.out.println(r2.compareTo(r3)); // 0

    Review[] badFruit = {r1, r2, r3, r4, r5};
    Arrays.sort(badFruit);
    for (int i = 0; i < badFruit.length; i++) {
        System.out.println(badFruit[i]);
    }
}
```

评论按评分排序输出：

```
Emoji Movie (10): Cinematic malware; likes = 0
Despicable Me 3 (60): Zany but scattershot humour; likes = 0
Dunkirk (95): Gifted ensemble cast and masterful direction; likes = 0
Spider Man: Homecoming (95): A fun adventure; likes = 0
My Neighbour Totoro (99): A work of art; likes = 0 
```

### 3.6.4. 实现 Comparable 还能做更多

除了支持排序之外，实现了 `Comparable` 的类还可以用于某些关心顺序的"集合（Collections）"，比如 `SortedSet`。关于集合，你将在后续阅读中学到。

## 3.7. Comparator 接口

如果我们想根据评分、评论文字长度或点赞数来选择不同的排序方式，该怎么办？一个类中只能有一个 `compareTo` 方法。或者，如果我们想让 `MonthDay` 只按月份比较，使得 2 月 14 日和 2 月 20 日被认为是相同的，又该怎么办？由于我们没有编写这个类，无法修改其 `compareTo` 的行为。

我们可以通过为每种比较方式定义一个"比较器（comparator）"类来实现这些目标。"比较器"类实现 `Comparator` 接口，该接口要求实现以下方法：

```java
int compare(T o1, T o2)    
    比较两个参数的顺序。
    当第一个参数小于、等于或大于第二个参数时，
    分别返回负整数、零或正整数。
```

以下是一个按点赞数对 `Review` 对象排序的 `Comparator` 示例：

```java
import java.util.Comparator;

class LikesComparator implements Comparator<Review> {
     /**
      * 比较两个参数的顺序。
      *
      * 当 r1 的点赞数小于、等于或大于 r2 时，
      * 分别返回负整数、零或正整数。
      *
      * @param r1 第一个要比较的 Review
      * @param r2 第二个要比较的 Review
      * @return 当 r1 小于、等于或大于 r2 时，返回负整数、零或正整数。
      */
    @Override
    public int compare (Review r1, Review r2) {
        return r1.getLikes() - r2.getLikes();
    }
}
```

（旁注：`compare` 方法需要知道评论的点赞数。由于这存储在私有实例变量中，我们添加了一个 getter 方法来提供访问。）

现在我们可以使用接受 `Comparator` 作为第二个参数的 `sort` 版本来决定排序方式。这里用 `LikesComparator` 调用它：

```java
public static void main(String[] args) {
    Review[] freshVeg = {r1, r2, r3, r4, r5};
    // 先加一些点赞，让排序结果更有意思。
    r1.like();
    r1.like();
    r1.like();
    r4.like();
    r3.like();
    Arrays.sort(freshVeg, new LikesComparator());
    for (int i = 0; i < freshVeg.length; i++) {
        System.out.println(freshVeg[i]);
    }
}
```

输出确实按点赞数排序：

```
Dunkirk (95): Gifted ensemble cast and masterful direction; likes = 0
Despicable Me 3 (60): Zany but scattershot humour; likes = 0
Spider Man: Homecoming (95): A fun adventure; likes = 1
My Neighbour Totoro (99): A work of art; likes = 1
Emoji Movie (10): Cinematic malware; likes = 3
```

我们还可以定义另一个 `Comparator`，按不同方式对 `Review` 对象排序。这个按文字长度排序：

```java
import java.util.Comparator;

class TextComparator implements Comparator<Review> {

    /**
     * 比较两个参数的顺序。
     *
     * 当 r1 的文字长度小于、等于或大于 r2 时，
     * 分别返回负整数、零或正整数。
     *
     * @param r1 第一个要比较的 Review
     * @param r2 第二个要比较的 Review
     * @return 当 r1 小于、等于或大于 r2 时，返回负整数、零或正整数。
     */
    @Override
    public int compare (Review r1, Review r2) {
        return r1.getText().length() - r2.getText().length();
    }
}
```

（这里我们需要为 `text` 实例变量添加一个 getter。）

现在可以使用 `TextComparator` 而不是 `LikesComparator` 对 `Review` 对象排序：

```java
public static void main(String[] args) {
    Review[] rottenVeg = {r1, r2, r3, r4, r5};
    r1.like();
    r1.like();
    r1.like();
    r4.like();
    r3.like();
    Arrays.sort(rottenVeg, new TextComparator());
    for (int i = 0; i < rottenVeg.length; i++) {
        System.out.println(rottenVeg[i]);
    }
}
```

输出按评论文字长度排序：

```
My Neighbour Totoro (99): A work of art; likes = 2
Spider Man: Homecoming (95): A fun adventure; likes = 2
Emoji Movie (10): Cinematic malware; likes = 6
Despicable Me 3 (60): Zany but scattershot humour; likes = 0
Dunkirk (95): Gifted ensemble cast and masterful direction; likes = 0
```

### 3.7.1. 何时使用 Comparable vs. Comparator？

如果你不是类的作者，就无法让它实现 `Comparable`，你唯一的选择是定义一个或多个比较器。

如果你是类的作者，两种方式都可以使用！
