# 第14章：正则表达式

## 学习目标

- 理解正则表达式（regex）的基本语法
- 判断给定字符串是否_符合_给定的正则表达式
- 在 Java 中应用正则表达式

## 14.1. 什么是正则表达式？

**正则表达式**（也称"regex"）是一个字符序列，用于描述一组字符串的模式。我们说一个正则表达式_匹配_所有满足该模式的字符串。反过来，如果一个字符串满足某个正则表达式描述的模式，则称该字符串_符合_该正则表达式。

以正则表达式 `.*o` 为例：
- `.` 匹配任意字符，
- `*` 表示前面的元素（这里是 `.`）出现零次或多次，
- `o` 匹配字符 `o`。

合在一起，这个完整的正则表达式匹配所有以字母 `o` 结尾的字符串，例如 "hello"、"ceo" 和 "o"。

如此示例所示，正则表达式为我们提供了一种更简洁的方式来描述可能任意大的字符串集合！让我们探索正则表达式的语法，以便理解并定义自己的模式。

## 14.2. 一些常用的正则表达式符号及示例

我们用 `[` 和 `]` 来定义一组可选符号。例如，`[abc]` 可以匹配 "a"、"b" 或 "c"。

通过插入连字符（`-`），可以指定一个范围。例如，`[a-z]` 匹配任意小写字母，如 "a"、"h" 或 "x"。

其他示例包括 `[0-9]` 或 `\d`，两者都匹配任意数字，如 "0"、"2" 或 "7"；`[a-zA-Z0-9]` 则可匹配 "b"、"R" 和 "2" 等字符串。

多伦多的邮政编码都以 `M` 开头，后面数字和字母交替出现，中间有一个空格。例如，"M0A 1B2" 就是一个多伦多邮政编码。要描述所有多伦多邮政编码，可以使用正则表达式 `M[0-9][A-Z] [0-9][A-Z][0-9]`。

## 14.3. 用量词重复模式

**量词**指定正则表达式中前面元素必须出现的次数，以使模式匹配成功。下表总结了这些量词：

| 量词       | 含义                  | 示例                                              |
|------------|----------------------|--------------------------------------------------|
| `*`        | 零次或多次             | `a*` 匹配空字符串、`a`、`aa` 等                   |
| `+`        | 一次或多次             | `a+` 匹配 `a`、`aa` 等                           |
| `?`        | 零次或一次             | `a?b` 匹配 `ab` 或 `b`                           |
| `{x}`      | 恰好 `x` 次           | `[ab]{2}` 匹配 `aa`、`ab`、`ba` 或 `bb`          |
| `{x,}`     | `x` 次或更多次         | `[a]{2,}` 匹配 `aa`、`aaa` 等                    |
| `{x,y}`    | `x` 到 `y` 次之间     | `[a]{2,4}` 匹配 `aa`、`aaa` 或 `aaaa`            |

### 14.3.1. 字符类

**字符类**是一组字符，如 `[a-z]`，用于指定可以出现的单个字符。集合中恰好有一个字符必须出现。

> 然后可以将上一节介绍的量词应用于字符类，以指定重复次数并创建更复杂的模式。

可以用方括号自定义字符类，如 `[q-z]` 和 `[AEIOU]`，也可以使用预定义的字符类。

下表列出了一些常用字符类：

| 字符类 | 描述                              |
|--------|-----------------------------------|
| `.`    | 任意字符（换行符除外）             |
| `\d`   | 数字 `[0-9]`                      |
| `\D`   | 非数字 `[^0-9]`                   |
| `\s`   | 空白字符                          |
| `\S`   | 非空白字符                        |
| `\w`   | 单词字符 `[a-zA-Z_0-9]`           |
| `\W`   | 非单词字符                        |

> 注意，`\w` 的定义包含 `_` 字符。

### 14.3.2. 分组与匹配重复字符的字符串

量词允许我们重复模式，但有时我们也希望重复_完全相同_的匹配字符。为此，需要用 `(` 和 `)` 指定一个**分组**。

例如，假设我们想要一个正则表达式，匹配形如 `xxx-xxx-xxxx` 且前三位数字相同的电话号码，如 "**222**-987-5406"、"**000**-232-3232"、"**111**-111-1114"。以下正则表达式使用分组来实现：

`([0-9])\1\1-[0-9]{3}-[0-9]{4}`

`\1` 表示分组 `1` 中的字符——这里是符合 `[0-9]` 的字符——必须重复出现。第二个 `\1` 表示还需要再重复一次，因此同一个字符必须连续出现三次！

### 14.3.3. 多个分组

每个分组根据其左括号从左到右出现的顺序被分配一个编号。因此，上例中唯一的分组是分组 `1`，在正则表达式中引用时写作 `\1`。

例如，`(([de])f)\2\1` 会重复两个分组。匹配的字符串为："dfddf" 和 "efeef"。

- 分组 1 对应模式中的 `[de]f` 部分
- 分组 2 对应模式中的 `[de]` 部分

为了更直观地看出匹配结果，可以用空格分隔写成：

`df d df` 和 `ef e ef`，从而清楚看出分组的重复。

## 14.4. 示例：Java 实例变量名的模式

IntelliJ 等 IDE 可能以如下方式描述 Java 变量的命名约定：

```^[a-z][a-zA-Z0-9]*$```

用普通语言描述，这个正则表达式的目的是：

"符合该正则表达式的字符串与 Java 变量命名约定一致"

让我们分析这个正则表达式中的符号，理解它是如何表达这个含义的！

`^` 字符表示模式必须从字符串的起始处开始匹配，这称为**锚点（anchor）**。

最左边的方括号中列出了所有小写字母，因此第一个字符必须是这 26 个字母之一。

第二个字符必须来自第二组方括号：`[a-zA-Z0-9]*`。

> 回顾：`*` 量词表示前面元素出现零次或多次。

最后，`$` 表示字符串的结束，这也是一个锚点。

综合来看，符合该正则表达式的字符串必须由一个 `a-z` 中的字母开头，后跟零个或多个小写字母、大写字母或数字。我们可以认出，这正是 Java 中合法变量名的定义！

有了这个正则表达式，可以用几个示例来验证：

以下字符串符合该正则表达式：

"x"、"numStudents"、"obj1"

以下字符串不符合：

"Alphabet"、"2ab"、"next_value"

> 以下字符串是否符合我们的正则表达式？
> "z3333"、"aBcB041"、"78a"

### 14.4.1. 没有锚点时会怎样？

上例中我们介绍了锚点：

- `^` 匹配字符串的开头。
- `$` 匹配字符串的结尾。

从理论上讲，正则表达式定义的是完全匹配该模式的字符串集合。然而在实践中，大多数正则表达式实现的行为有所不同——除非使用锚点，否则它们通常检查输入的某个_子字符串_是否符合模式。

> 在实践中，在 Java 等编程语言中使用正则表达式时，必须仔细阅读文档，因为有些方法会隐式包含锚点（即该方法会自动查找完全匹配）。

下面是一个没有锚点的正则表达式：

```[abc]C[a-e][24680]*[A-Z]```

由于没有锚点，只要字符串中包含符合该正则表达式的_任意子字符串_，该字符串就符合此正则表达式。

例如，"cCaA" 符合，"ABCcCcAa1A23" 也符合，因为它们都包含子字符串 "cCcA"，与该正则表达式一致。

但 "cCa1A" 不符合，因为其中的 "1" 与模式不一致。

## 14.5. 特殊符号

正则表达式中某些字符具有特殊含义，使我们能够开发更丰富的表达式。

例如，正如我们已经看到的，句点（`.`）匹配任意字符。

在处理文件和字符串时，你可能记得反斜杠（`\`）转义字符可以与某些其他字符结合使用，表示换行符（`\n`）或制表符（`\t`）等。在正则表达式中也是如此。预定义字符类（如 `\s` 和 `\d`）也是这方面的示例。

在方括号内部，`^` 还有另一种含义——不再是锚点，而是匹配方括号内容_以外_的任意字符。

例如，`[^aeiouAEIOU]` 匹配任何_不是_元音字母的字符。

### 14.5.1. 转义符号的特殊含义

如果我们想在符合的字符串中包含一个句点，该怎么办？要匹配 "1.2" 或 "3.8" 这样的字符串，需要某种方法强制包含句点。由于 `.` 匹配任意字符而不仅仅是句点，我们需要在句点前加反斜杠来**转义**它：`\.`。通过转义句点，表明其特殊含义应被忽略，`\.` 精确匹配字面量 "." 字符。

因此，匹配所有个位数后跟小数点再跟一位数字的正则表达式如下：`[0-9]\.[0-9]`

### 14.5.2. 逻辑运算符

`|` 符号表示"或"，`&&` 表示其前后范围的交集。

例如，`[a-t&&r-z]` 只匹配 "r"、"s" 和 "t"。

> 注意：不同的正则表达式实现可能支持略有不同的逻辑运算符。

## 14.6. Java 中的正则表达式

在 Java 中，正则表达式以字符串形式书写，因此需要对所有反斜杠进行转义。本来写 `\d` 匹配单个数字，在 Java 中需要写成 `"\\d"`。这也意味着，转义一个反斜杠本来写作 `\\`，在 Java 中要写成 `"\\\\"`, 因为每个反斜杠都需要单独转义。

`String` 类提供了 `split`、`matches`、`replaceAll` 和 `replaceFirst` 等方法，阅读其文档可以发现，这些方法都使用了正则表达式。

[`Pattern`](https://docs.oracle.com/javase/8/docs/api/java/util/regex/Pattern.html) 类用于表示正则表达式。它提供静态方法 `compile(String regex)` 和 `matches(String regex, String string)`，分别用于根据给定正则表达式字符串创建 Pattern 对象，以及将给定正则表达式与给定字符串进行匹配。建议阅读上方链接的文档，其中对 Java 中的正则表达式语法有很好的总结。

例如，`Pattern` 文档提供了如下基本用法示例：

```java
Pattern p = Pattern.compile("a*b");
Matcher m = p.matcher("aaaaab");
boolean b = m.matches();
```

文档还指出，`matches` 提供了一种更简便的等价写法：

```java
boolean b = Pattern.matches("a*b", "aaaaab");
```

[`Matcher`](https://docs.oracle.com/javase/8/docs/api/java/util/regex/Matcher.html) 类用于执行与实际匹配检查及查询结果详情相关的各种操作。例如，可以使用 `group(int group)` 方法访问上次匹配操作中的某个分组。更多详情请参阅文档。

我们还提供了 [`Demo.java`](code/regex/Demo.java)，其中包含更多示例。

## 14.7. 正则表达式能做什么？

如我们所见，正则表达式提供了简洁而富有表达力的方式来描述文本中的模式。它们允许我们执行各种任务，例如：

- 在文档中查找数据
  - 示例：查找文件中的电话号码、网页上的电子邮件地址、文本中的 CSC 课程代码，或在 IntelliJ 中使用"查找和替换"功能。

- 验证输入
  - 示例：检查变量名是否符合 Java 命名约定，或密码是否满足复杂度要求。

总体来说，这些任务分为两类：
- **提取**：找出符合正则表达式指定模式的子字符串。
- **验证**：判断整个字符串是否符合某个模式。

正则表达式在编程语言和工具中得到了广泛支持，是开发者必须掌握的基本技能。
